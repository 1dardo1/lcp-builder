## 9. Ammo


**Fuente:** Weapons (sección Ammo del README/página) — verificado sin discrepancias entre la versión vista en `weapons` y fuentes adicionales.

> **Nota de reutilización:** igual que `Synergies` (sección 5), `IAmmoData` usa los enums "reales" `WeaponType`/`WeaponSize` (sección 1), no los filtros de `Bonuses`. Mismo mecanismo de "omitir = todos los tipos/tamaños" que en `Synergies`; las restricciones (`restricted_types`/`restricted_sizes`) tienen precedencia sobre las permisiones (`allowed_types`/`allowed_sizes`). **Importante:** estos mismos nombres de campo (`restricted_types`/`restricted_sizes`) aparecen también en `IWeaponModData` (sección 13.5), pero ahí están marcados explícitamente como deprecados por la fuente (mantenidos solo por compatibilidad v2). En `Ammo`, la página no los marca como deprecados — se documentan aquí como vigentes. No es un error de transcripción a corregir; son dos contextos distintos de la spec con estados de vigencia distintos para el mismo nombre de campo.

### IAmmoData
**Tipo:** Value Object

```
IAmmoData: {
  name: string,                      // requerido
  description: string,               // requerido, v-html
  cost?: number,                     // default 1
  allowed_types?: WeaponType[],      // omitir = aplica a todos los tipos
  allowed_sizes?: WeaponSize[],      // omitir = aplica a todos los tamaños
  restricted_types?: WeaponType[],   // tiene precedencia sobre allowed_types
  restricted_sizes?: WeaponSize[]    // tiene precedencia sobre allowed_sizes
}
```
