## 3. Contenedores de efectos


> **Nota de uso — Action vs. ActiveEffect:** usar `Action` cuando el efecto tiene un coste de activación explícito según las reglas (aunque ese coste sea una Acción Gratuita). Usar `ActiveEffect` cuando el efecto es pasivo, no tiene flujo de activación propio, o aplica fuera del ciclo de turnos (aunque pueda tener una condición que lo activa).
>
> **Asimetría de anidación:** `IActionData` puede contener `IActiveEffectData` (campo `active_effects`). `IActiveEffectData` NO puede contener `IActionData` — no existe campo de ese tipo. La anidación es unidireccional: Action → ActiveEffect, nunca al revés.

### ActivationType (enum)
**Fuente:** Actions / README — enum base sin dependencias, definido al principio de la sección porque `IActionData` (más abajo) lo necesita.

```
enum ActivationType { Free, Protocol, Quick, Full, Invade, QuickTech, FullTech, Reaction }
// Strings JSON exactos (case-sensitive): "Free", "Protocol", "Quick", "Full",
// "Invade", "Quick Tech", "Full Tech", "Reaction"
```

### IActiveEffectData
**Tipo:** Value Object
**Fuente:** Active Effects

```
IActiveEffectData: {
  name: string,
  detail: string,
  condition?: string,
  frequency?: ActionFrequency,
  duration?: EffectDuration,
  bonus_damage?: DiceExpression,
  damage?: IDamageData[],
  range?: IRangeData[],
  add_status?: IStatusEffectData[],
  add_resist?: IResistanceData[],
  add_special?: ISpecialStatusData[],
  remove_special?: string[],
  add_other?: IOtherEffectData[],
  save?: IEffectSaveData,
  attack?: enum { melee, ranged, tech },
  pilot?: boolean,
  mech?: boolean,
  accuracy?: number,
  attack_bonus?: number
}
```

### IActionData
**Tipo:** Value Object
**Fuente:** Actions

```
IActionData: {
  name: string,                  // requerido (excepción: heredado del item padre en algunos systems)
  activation: ActivationType,    // enum: Free, Protocol, Quick, Full, Invade, Quick Tech, Full Tech, Reaction
  detail: string,
  frequency?: ActionFrequency,
  trigger?: string,               // requerido SI activation = Reaction
  cost?: number,                  // default 1 si el item es "limited"
  pilot?: boolean,
  mech?: boolean,
  hide_active?: boolean,
  bonus_damage?: DiceExpression,
  damage?: IDamageData,           // singular — distinto de IActiveEffectData.damage (array)
  range?: IRangeData,             // singular — distinto de IActiveEffectData.range (array)
  add_status?: IStatusEffectData[],
  add_resist?: IResistanceData[],
  add_special?: ISpecialStatusData[],
  remove_special?: string[],
  add_other?: IOtherEffectData[],
  active_effects?: IActiveEffectData[],
  save?: IEffectSaveData
}
```
