## 1. Tipos primitivos compuestos (hojas del árbol de dependencias)


> **Criterio de orden dentro de esta sección:** de menor a mayor complejidad interna — primero los enums (conjunto fijo y cerrado de valores, sin gramática propia), después `DiceExpression` (un string con reglas de formato y evaluación propias). El orden NO refleja la secuencia en que se descubrieron durante la investigación de la spec, sino la complejidad real de cada tipo.

### DamageType (enum)
**Fuente:** Damage

```
enum DamageType { Kinetic, Energy, Explosive, Heat, Burn, Variable }
```

### RangeType (enum)
**Fuente:** Range

```
enum RangeType { Threat, Range, Burst, Blast, Cone, Line }
// Nota: el valor "Range" del enum coincide en nombre con el tipo
// contenedor IRangeData. No representa conflicto si se nombra
// el valor del enum en minúscula/distinto en Dart (ej. `range`).
```

### WeaponType (enum)
**Fuente:** Weapons — definido aquí, fuera de su bloque temático, porque `Bonuses` (sección 4), `Synergies` (sección 5) y `Ammo` (sección 9) ya lo necesitan antes de llegar al bloque completo de Weapons.

```
enum WeaponType { Rifle, Cannon, Launcher, CQB, Nexus, Melee }
// "CQB" verificado contra lib/weapons.json (sección 4) — la grafía "CQC",
// vista en la prosa de la página de Bonuses, es un error de tecleo de esa
// página, no un valor real.
```

### WeaponSize (enum)
**Fuente:** Weapons — mismo motivo que WeaponType, definido aquí por necesidad anticipada.

```
enum WeaponSize { Auxiliary, Main, Heavy, Superheavy }
// "Auxiliary" verificado contra lib/weapons.json (campo mount, ej.
// mw_missile_rack: "mount": "Auxiliary") — la grafía "Aux", vista
// originalmente en el README inicial y en la página de weapons, es
// incorrecta para este campo. Distinto del valor "Aux" en MountAssignment
// (sección 4), que es un campo distinto de la spec con su propia grafía
// verificada por separado.
```

### DiceExpression
**Tipo:** Value Object
**Fuente:** Damage, Range (definición compartida)

Representa una expresión de daño/rango/valor. Es `number` o `string`. Como string, sigue el patrón:

```
DiceExpression = number | string

// Como string, la forma es: XdY+Z, XdY, o Z, donde:
//   X = número de dados (opcional, default 1)
//   Y = caras del dado (ej. 6, 20)
//   Z = bonus plano (opcional, default 0)
// El valor completo se evalúa como expresión aritmética normal:
// admite +, -, *, /, paréntesis, y "bonus strings" entre llaves
// que se resuelven en tiempo de ejecución (ej. {grit}, {ll}, {agi}).
// Ejemplos válidos: "10", "2d6+4", "(1d8+2)*2", "2+1d6+{agi}"
//
// NOTA DE ALCANCE: la validación/evaluación completa de esta expresión
// (parsing de la gramática aritmética + resolución de bonus strings) es
// lógica de tiempo de ejecución, no una validación de construcción simple.
// Para la fase de creación/exportación, basta con almacenar el string
// y validar el patrón básico (XdY+Z); la evaluación completa puede
// posponerse a una fase posterior.
```

### TargetType (enum)

Tipo transversal: no es exclusivo de Damage/Range, también se reutiliza en los subtipos de Active Effects (sección 2). Se define aquí porque `IDamageData` (más abajo, en esta misma sección) ya lo necesita — es el punto más temprano del documento donde aparece. El valor por defecto cuando se omite **varía según el tipo contenedor que lo usa** — se indica en cada uno, no aquí.

```
enum TargetType { self, ally, enemy }
```

### IDamageData
**Tipo:** Value Object
**Fuente:** Damage

```
IDamageData: {
  type: DamageType,
  val: DiceExpression,
  aoe?: string | boolean,
  save?: string | { stat: string, aoe?: boolean },
  save_half?: boolean,
  ap?: boolean,
  target?: TargetType  // default 'enemy'
}
```

### IRangeData
**Tipo:** Value Object
**Fuente:** Range

```
IRangeData: {
  type: RangeType,
  val: DiceExpression,
  min?: DiceExpression
}
```
