## 2. Subtipos de Active Effects


**Fuente:** Active Effects

> **Criterio de orden:** mismo principio que la sección 1 — los tipos pequeños y reutilizados van antes de cualquier tipo que los use, aunque en la conversación se hayan descubierto "dentro" de otro tipo más grande. `TargetType` se define en la sección 1 (la necesita `IDamageData`) y se reutiliza aquí sin redefinirse.

### EffectDuration (enum especial)

```
EffectDuration =
  | 'next_turn_start_self'
  | 'next_turn_end_self'
  | 'next_turn_start_target'
  | 'next_turn_end_target'
  | 'round_start_X'   // X = número de ronda entero, parte variable del string
  | 'round_end_X'     // X = número de ronda entero, parte variable del string
```

### MechStat (enum)

Reutilizado en `IStatusEffectData.save`, `IEffectSaveData.stat`. Las cuatro estadísticas base del mech sobre las que se puede tirar salvación.

```
enum MechStat { hull, agi, sys, eng }
```

### IStatusEffectData
**Tipo:** Value Object — `id` referencia un catálogo compartido (statuses.json), no identidad de instancia.

```
IStatusEffectData: {
  id: string,              // debe existir en core data o LCP statuses.json cargado
  duration?: EffectDuration,
  save?: MechStat,
  aoe?: string | boolean,
  target?: TargetType   // default: enemy
}
```

### IResistanceData
**Tipo:** Value Object — union discriminado por exclusión mutua.

> **Corrección verificada contra JSON real:** la página de Active Effects documenta el campo como `resistance`, pero `lib/frames.json` usa `resist` en todas las apariciones reales comprobadas (Drake "Blast Plating": `{"resist": "aoe"}`; Black Witch "Repulsor Field": `{"resist": "Kinetic"}`). Se usa `resist`, el valor real, no `resistance`, el de la prosa de la Wiki. `immunity` sí coincide entre ambas fuentes.
>
> **Pendiente — `vulnerability` no confirmado:** no se encontró ningún ejemplo real de este campo en la muestra de JSON revisada hasta ahora. Se mantiene el nombre `vulnerability` por inferencia de simetría con `resist` (ambos documentados en la misma página, con la misma estructura), pero **no está verificado contra datos reales**. Si aparece evidencia en otro archivo (p. ej. `npc_features.json`), revisar y corregir si difiere.

```
type IResistanceData =
  | { resist: ResistanceValue;         vulnerability?: never; immunity?: never; target?: TargetType }
  | { vulnerability: ResistanceValue;  resist?: never;        immunity?: never; target?: TargetType }
  | { immunity: ImmunityValue;         resist?: never;        vulnerability?: never; target?: TargetType }

// resist/vulnerability deben ser uno de:
//   kinetic | energy | explosive | heat | burn | aoe | all
// immunity admite los mismos valores, o cualquier id de Status/Condition
// (core data o LCP statuses.json cargado).
type ResistanceValue = enum { kinetic, energy, explosive, heat, burn, aoe, all }
type ImmunityValue = ResistanceValue | string  // string = id de Status/Condition

// target default: self (distinto del default de IStatusEffectData)
```

### ISpecialStatusData
**Tipo:** Value Object

```
ISpecialStatusData: {
  attribute: string,        // requerido — terse, una línea en UI
  detail?: string,          // sin HTML; segunda persona presente
  target?: TargetType,      // default: enemy
  duration?: EffectDuration
}
```

### IOtherEffectData
**Tipo:** Value Object — union discriminado por `type`.

```
type NumericOrFormulaValue = number | string  // número fijo o fórmula con bonus syntax, ej. "{grit} + 4"
type CoverLevel = 'soft' | 'hard' | 'none'

type IOtherEffectData =
  | { type: 'overshield'; val: NumericOrFormulaValue; target?: TargetType; aoe?: string | boolean }
  | { type: 'hp';         val: NumericOrFormulaValue; target?: TargetType; aoe?: string | boolean }
  | { type: 'repair';     val: NumericOrFormulaValue; target?: TargetType; aoe?: string | boolean }
  | { type: 'cover';      val: CoverLevel;            target?: TargetType; aoe?: string | boolean }

// target default: ally
```

### IEffectSaveData
**Tipo:** Value Object

```
IEffectSaveData: {
  stat: MechStat,
  aoe?: string | boolean
}
// Forma corta admitida: un string suelto 'hull'|'agi'|'sys'|'eng',
// equivalente a { stat: string, aoe: false }
```

### ActionFrequency (enum)
**Fuente:** Actions, Active Effects — conjunto fijo, SIN variable entera.

```
ActionFrequency =
  | 'unlimited'      // default
  | '1/round'
  | '1/turn'
  | '1/scene'
  | '1/encounter'    // alias de '1/scene'
  | '1/mission'
```

### ReactionFrequency (enum especial)
**Fuente:** Weapons (campo `frequency` de Reactions) — distinto de ActionFrequency: admite X entero variable.

```
ReactionFrequency =
  | 'Unlimited'
  | 'X/round'       // X = entero positivo
  | 'X/scene'
  | 'X/encounter'   // alias de X/scene
  | 'X/mission'
// Cualquier otro valor se trata como 'Unlimited' en la UI.
```
